文體集

昭源字體配置了若干文體集 (stylistic sets)，讓使用者可以按需要選擇不同的字形風格。

備註

文體集功能目前處於測試階段，可能會有變動。

ss15

Section titled “ss15”

Unicode 為了兼容此前各地字碼表，採取了「字源分離原則」，將一些本應整合、但因為原來字碼表已經分成兩個字碼的字形編配不同的碼位。以「昷」部件為例，另一個常見寫法是「𥁕」。二者顯然是同一部件的變體。雖然昭源字體偏好「昷」形，但此部件不少常用字是將「從『𥁕』」、「從『昷』」形態分開編碼，於是「溫温」「搵揾」「蘊藴」等字從「囚」抑或從「日」，就不能按字體設計者的偏好，而須按使用者輸入了哪一個字碼決定字形。

但是「字源分離原則」只適用於 Unicode 初創時要整合的字碼表中已經分離的字形。同樣是從「昷/𥁕」部件的字，「瘟」卻由於不須採用「字源分離原則」，只有一個碼位 ( U+761F)。這時候輸入這個字碼，呈形出來的字形到底是從「囚」抑或從「日」，就是字體設計者的決定，屬於字體風格問題。昭源字體偏好「昷」寫法，但基於字源分離原則不能將「溫搵蘊」做成「温揾藴」，結果就是假如使用者習慣輸入「溫搵蘊」等字，與「瘟」字同排時，就會出現不一致的字形風格。

為何不乾脆將從「𥁕」部件字碼全部做成「昷」形？

雖然技術上可行，但可能會造成資訊處理的混亂。例如有人寫「『溫』『温』哪一個寫法才對？」，字體將二字做成同一形，使用者用這款字體顯示的句子就會變成「『温』『温』哪一個寫法才對？」，令人不明所以。

昭源作為一款通用字型，預設不會將「字源分離原則」的字形統一，但透過 ss15 文體集，讓使用者按需要，將一些本應整合、只是礙於字碼表已經分成兩個字碼的字形劃一寫法。

備註

ss15 文體集也包含一些非因「字源分離原則」而分開，但幾無爭議的可統一全等異體字。又統一字形主要處理 URO 區的漢字，首先針對「較容易在繁體中文環境輸入成異體（非本字體偏好字樣）」的情況，再適度擴充。本字體無意將全部可統一字碼納入文體集，文體集用意也不是為了自動改正錯寫。

這個功能有助於例如在有編輯校對一般性排版時，可以避免容易看漏眼、不同風格的字形混用，提高排版的一致性。當然，如果文章內容是有關字形寫法的討論，又或是例如偵探小説而破案關鍵與字形有關，則不宜使用此文體集。
